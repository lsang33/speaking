Section 01

快速开始

1
获取 API Key
前往任意支持的 AI 服务商注册账号，获取 API Key。推荐国内用户使用 DeepSeek（价格低廉）或 Moonshot Kimi，国际用户使用 Anthropic Claude 或 OpenAI。
2
打开 app.html
直接用浏览器打开 app.html 文件即可在 Mac/PC 上使用。iPhone 需要部署到 HTTPS 地址（见"iPhone 部署"章节）。
3
填入 API Key
首次打开会出现设置页，输入 API Key 后点击"开始使用"。Key 只存在本地设备，不上传。默认使用 Anthropic，可以在设置里切换其他服务商。
4
选择模式，开口说
首页有两个训练入口：开始练习（按题目说）和词语串联（把随机词语串成一段话）。选一个，给自己几秒准备，然后开口录音。
5
获取 AI 评估
点击"完成，获取 AI 评估"，等待 10 秒左右，AI 会给出结构分析、亮点、改进建议和综合评分（1-10 分）。
💡 目标时长

每次练习目标 90–210 秒（1.5–3.5 分钟）。不足 90 秒会被计为偏短。哪怕今天只能说 60 秒，也比不说强——坚持比完美重要。

两种训练模式

🎯 按题练习
基础模式

系统从题库中抽取一道题目，你有准备时间思考，然后录音作答。


题库涵盖 8 大主题，共 40+ 道题：

8 题

6 题
家国情怀
5 题
哲理思辨
6 题
个人成长
5 题
职场观察
4 题
未来想象
若干

每道题都附带思路参考，列出 3 个思考角度，帮你快速搭建结构。不喜欢当前题目可以点换题重新抽取。


难度从 ① 简单（个人经历）到 ③ 较难（宏观热点），每天 APP 会根据日期分配一道"今日题目"，也可随时换题。

🔀 词语串联
快反训练

随机出现 2–5 个词语，你需要在准备时间内想好思路，然后把这些词语自然地串进一段完整表达里。


词库 120+ 个词，覆盖：日常物件（镜子、钥匙、闹钟…）、自然意象（彩虹、落叶、星空…）、情感抽象（勇气、遗忘、释然…）、社会概念（边界、机遇、惯性…）、隐喻物象（灯塔、沙漏、棱镜…）等多个类别。


录音时，屏幕上方显示词语进度条。你每说出一个词，它就变绿打勾，让你实时知道还差哪个词没用上。


AI 评估会额外考察：词语覆盖率（说了几个）、词语融入是否自然（硬塞 vs. 浑然天成）。


不必按词语的顺序说，也不需要解释词语含义——只要让它们出现在你的故事、观点或场景里就行。

🎓 两种模式配合使用
  • 按题练习适合主要训练：逻辑结构、论据积累、主题表达
  • 词语串联适合专项训练：临场反应、联想能力、语言灵活性
  • 建议每周 5 天"按题"＋2 天"词语串联"交替
Section 03

完整练习流程

📋
① 查看题目 / 词语
按题模式看题目和思路参考；词语模式看随机词组。不满意可点"换题"或"换一组"重新抽取。
⏳
② 准备倒计时
默认 30 秒准备时间，在脑中快速搭框架。倒计时归零后自动进入录音。也可随时点"现在开始"跳过。
3️⃣
③ 3-2-1 开始
进入 3 秒倒数，给你一口深呼吸的时间。倒数结束自动开始录音，不需要额外操作。
🎙
④ 录音中
说话，文字实时转写在屏幕上滚动（用于 AI 分析）。词语串联模式会实时高亮已说出的词。
🤖
⑤ AI 评估
点"完成"后 AI 结合文本+音频数据生成评估报告，约 5-15 秒。无 API Key 或网络问题时会提示手动输入文本。
📈
⑥ 复盘结果
查看各维度反馈、本次亮点、明日聚焦建议，以及 1-10 综合评分。记录自动存入历史档案。

🔇 语音识别不可用时

如果设备不支持语音识别（或未在 HTTPS 环境下），录音结束后会弹出手动输入框，可将自己说的内容大致输入，AI 仍可据此评估逻辑和流畅度。音频文件照常保存可以用于回听复盘。

Section 04

AI 评估机制

评估结合两类数据：程序精确统计的客观指标，和 AI 对文本内容的语义分析。评分以客观数据为优先约束，AI 不会无故虚高。

📊 客观数据指标

发言时长 目标 90–210 秒
有效字数 中文字符数
语速 正常 200–270 字/分
口头禅 嗯/啊/就是/然后 等 12 个
停顿次数 ≥0.5 秒有效停顿
音量变化 单调 vs 有起伏

📝 AI 评估维度

逻辑结构
有无完整的"观点→论据→升华"层次，结构是否清晰完整
语言流畅度
结合口头禅次数评价，会点名具体是哪个词出现过多
本次亮点
直接引用你说的最精彩的一句原话
明日聚焦
最应优先改进的一点，并给出具体可操作的做法
词语串联
仅词语模式
哪些词用得自然、哪些生硬或遗漏，以及词语覆盖率

⚖️ 评分锚点（严格参照，不虚高）

9–10 逻辑完整（观点→论据→升华），口头禅 0 次，语速适中，有感染力，时长达标
7–8 逻辑清晰，口头禅 ≤2 次，能完整表达观点，时长基本达标
5–6 有基本观点但逻辑松散，口头禅 3–5 次，或时长不足 90 秒
3–4 思路混乱，口头禅 ≥6 次，或严重偏题，或时长极短
1–2 内容极少、无关主题或无法理解
💡 如何提高评分
  • 开口前 10 秒内亮明核心观点，不要绕弯子
  • 用 PREP 结构：观点 → 理由 → 例子 → 重申
  • 口头禅是最拖分的单项，刻意减少"然后然后然后"
  • 停顿不是弱点——有意义的停顿加分，乱停顿才扣分

按"生活观察""社会热点""词语串联"等标签过滤记录，方便专项回顾。
🔊
录音回听
每条记录展开后可直接在应用内播放原始录音，方便对照 AI 反馈做复盘。

🗂 记录详情（点击展开）

  • 发言时长、日期、主题
  • 词语串联记录：各词语是否被覆盖（绿色=已说，灰色=遗漏）
  • AI 评估全文：各维度分析 + 鼓励语
  • 转写原文：语音识别的完整文字记录
  • 单条导出录音（↓ 按钮）/ 删除此记录

📤 导出录音

历史页右上角有"导出录音"按钮，可批量导出当前筛选下的所有录音：

  • Mac / PC：弹出目录选择器，自动在所选位置创建 每日说/YYYY-MM/ 子文件夹，批量写入文件
  • iPhone：逐条触发下载，会落入"文件"App 的下载文件夹
  • 文件命名格式：每日说_YYYY-MM-DD_主题.mp4

💾 存储说明

所有数据存储在浏览器的 IndexedDB 中，完全本地，不上传服务器。历史页底部显示当前占用空间。清空数据的入口在"设置"页最底部。

兼容 OpenAI 格式的任意服务商，需填 API 地址 + Key + 模型名

服务商 推荐模型 说明

⏱ 准备时间

题目出现后的准备倒计时，默认 30 秒。可选：不限时（0）/ 15秒 / 30秒 / 1分钟 / 2分钟，或手动输入任意秒数。设为 0 则不显示倒计时，直接显示手动开始按钮。

🔀 词语串联默认词数

设置进入词语串联模式时默认抽取的词语数量（2–5 个）。也可以在词语串联页面顶部实时切换，不影响当前练习状态。

Section 07

iPhone 部署

⚠️ iPhone Safari 的语音识别 API 要求 HTTPS 环境。直接用文件路径打开 app.html 不支持录音转写，但录音本身可以正常工作，评估时需手动输入内容。

推荐两种免费 HTTPS 部署方式：

A
GitHub Pages（推荐，永久免费）
1. 在 GitHub 新建一个仓库（建议设为 Private）
2. 上传 app.html，在仓库设置里开启 GitHub Pages
3. 访问 https://你的用户名.github.io/仓库名/app.html
4. 在 Safari 里打开该链接 → 分享 → 添加到主屏幕，即可像 App 一样使用
B
Netlify Drop（最快，拖拽即部署）
1. 打开 app.netlify.com/drop
2. 把 app.html 直接拖入页面
3. 几秒钟后得到一个 HTTPS 链接（如 xxx.netlify.app）
4. 同样添加到主屏幕即可
⚠️ 不登录账号的情况下链接 1 小时后失效，注册免费账号可永久保留
📱 添加到主屏幕后的效果
  • 全屏沉浸式界面，没有地址栏和工具栏
  • 状态栏深色透明，融入 App 设计
  • 支持录音权限请求（第一次打开时授权）
  • 数据存在 Safari 的 IndexedDB 中，清除 Safari 缓存会丢失记录——建议定期导出录音备份
Section 08

进阶技巧

🧠 快速搭框架：即兴四步

  1. 描述——我看到/听到/遇到了什么？（具体，不抽象）
  2. 联想——这让我想到什么？（个人的 / 社会的 / 历史的）
  3. 深意——它背后说明了什么道理？
  4. 金句——用一句话提炼总结

🎯 词语串联高分策略

  • 先看所有词语，找 1–2 个能串联其他词的"主题词"
  • 用叙事类开头往往更容易自然地嵌入多个词，如"我曾经遇到过这样一件事…"
  • 不要堆砌——把词用在转折或升华处，而不是罗列解释
  • 遗漏的词不如强行放入，覆盖率是硬指标

📅 打卡习惯建议

  • 保底原则：再忙也录一条 60 秒，不能因为没时间做完整练习就跳过
  • 每月对比：每月第一天，听一条一个月前的录音和今天的对比，这是最真实的进步感知
  • 一周只盯一个点：本周专攻"减少口头禅"，下周再看逻辑结构，聚焦比全面更有效
  • 亮点也值得记：AI 说的"本次亮点"如果特别好，截图存下来，这是你的语言资产

🔧 技术要点

  • 录音格式：iPhone 上保存为 .mp4，其他设备为 .webm
  • 语音识别：使用设备内置的 Web Speech API，无需联网，完全本地
  • 音频分析：本地 Web Audio API 处理，检测停顿和音量变化，不上传音频
  • API 调用：仅评估文本时向你配置的 AI 服务商发送请求，不发送音频文件
  • 隐私：API Key、所有录音和历史记录均仅存储在本地设备
